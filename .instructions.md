2. Conventions

- Classes: PascalCase
- Files for classes: PascalCase.js
- Commands/events: camelCase.js
- Methods/variables: camelCase

3. Core

- Bot.js: orchestrator, uses container, starts Discord client, registers handlers
- Container.js: DI container, provides dependencies
- CommandHandler.js: loads commands, executes on interactions
- EventHandler.js: loads events, attaches to Discord client

4. Modules

- Each module: commands/, services/, events/
- Commands are controllers, services contain logic
- Inject dependencies, do not instantiate Discord client inside module

5. Commands & Events

- Commands: { name, description, execute() }
- Events: { name, once, execute(client) }
- Handlers auto-load them

6. Workflow

1) Core classes (Container, Bot, Handlers)
2) Basic commands (ping, help)
3) First module (music)
4) Add commands/services
5) Extend modules

7. Practices

- DRY, KISS, SoC, OOP, modular
- Core never depends on modules
- Use dependency injection everywhere
- OOP where it makes sense, not necessarily for everything
